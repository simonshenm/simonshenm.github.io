专题十三:元胞自动机

2D元胞自动机:冯诺依曼和摩尔型

一.火灾模型/渗流模型

1.模型假设说明

  ⋅\cdot⋅在一片森林中随机分配着多个数目
  ⋅\cdot⋅某些树着火会引起临近的树也着火
  ⋅\cdot⋅初始的述目密度严重影响火灾的大小

2.建立模型

  ⋅\cdot⋅每个方格有三种状态：空地、着火、树木
  ⋅\cdot⋅冯诺依曼型邻居：4个邻居
  ⋅\cdot⋅*状态变换
下面给出netlogo程序

to setup
clear-all
ask patches with[(random-float 100) < density]
[set pcolor green]
ask patches with[pxcor = min-pxcor]
[set pcolor red]
end

to go
ask patches with[pcolor = green]
[
if(count neighbors4 with [pcolor = red] > 0)
[
set pcolor red
]
]
tick
end


  发现在density在60左右处于一种临界状态，处于一个分形几何体，考虑一种突变函数，叫做渗流，也类似sigmoid函数；density过小会烧不起来；density过大会一下子就烧完了。

二.沙堆模型

1.模型假设说明

  ⋅\cdot⋅能量缓慢积累
  ⋅\cdot⋅在某一瞬间突然释放
  ⋅\cdot⋅系统中会出现大大小小的雪崩
  ⋅\cdot⋅自组织临界（系统呈现临界状态，系统会自发演化并维持这种临界状态）

2.建立模型

  ⋅\cdot⋅是冯诺依曼邻居型
  ⋅\cdot⋅五个状态（五个格子，放不同数量的沙子，1，2，3，4，5粒）
下面给出netlogo程序

pathces-own [myvalue]
to setup
ask patchs[
set myvalue 0
colorit
]
end

to go
ask one-of patches[
set myvalue myvalue+1
]
ask patches[
judge-cascading
]
ask patches[
colorit
]
tick
end

to colorit
if(myvalue = 0)
[
set pcolor black
]
if(myvalue = 1)
[
set pcolor blue
]
if(myvalue = 2)
[
set pcolor green
]
if(myvalue = 3)
[
set pcolor red
]
if(myvalue = 4)
[
set pcolor yellow
]
end

to judge-cascading
if(myvalue > 4)
[
set myvalue myvalue - 4
ask neighbors4[
set myvalue myvalue + 1
]
]
end


  ⋅\cdot⋅沙粒一粒一粒往下放是能量积累的过程
  ⋅\cdot⋅小的崩塌时常发生，而大的崩塌偶尔发生，类比于地震，社会系统

一.投票模型

1.模型假设说明

  ⋅\cdot⋅一个人工村落
  ⋅\cdot⋅每个人有8个邻居
  ⋅\cdot⋅村民对总统竞选人进行投票
  ⋅\cdot⋅邻居观点会对此人的选择实行影响
  ⋅\cdot⋅考察这些人的观点如何随着时间改变

2.建立模型

  ⋅\cdot⋅每个格点有m种状态
  ⋅\cdot⋅随机选择一个格点拷贝它的状态
  ⋅\cdot⋅是一个随机元胞自动机，其实就是一个多数原则

下面给出netlogo程序

patches-own[
value
]

to setup
ca
ask patches
[
set value random options
colorit
]
end

to go
ask n-of number-changes patches[
set value [value] of one-of neighbors
]
ask patches
[
colorit
]
do-plot
end

to colorit
let colors[black green blue yellow red brown sky violet orange pink]
set pcolor(item value colors)
end

to do-plot
set-current-plot “options”
let i 1
let species 0
while(i < options)
[
let num(count patches with[value = i])
if(num > 0)
[
set species species + num
set i i + 1
]
plot species
]
end

四.其它元胞自动机

1.按照空间排列方式

  ⋅\cdot⋅四角格
  ⋅\cdot⋅三角格
  ⋅\cdot⋅六角格（用的多，各向同性，沿着任何一个方向概率相等，圆形）

五.自组织临界

1.说明

  ⋅\cdot⋅系统处于临界状态，就是4，4，4，5那种情形
  ⋅\cdot⋅崩塌会产生级联效应
  ⋅\cdot⋅崩塌的时空呈现幂律分布
  ⋅\cdot⋅系统自发组织进入临界态

1D元胞自动机

一.基本说明

  ⋅\cdot⋅在一条1D的直线上
  ⋅\cdot⋅每个格子的左右两侧半径为r内的方格为其邻居
  ⋅\cdot⋅每个方格的状态可以用不同的颜色来表示
  ⋅\cdot⋅规则为一个列表

二.1D元胞自动机的运行

  ⋅\cdot⋅将某一时刻所有格子排成一排
  ⋅\cdot⋅所有时刻的格子按照时间先后顺序从上到下进行排列得到一张2D图

matlab代码和仿真模拟图



%%
%元胞自动机，燃烧森林模型
% 2016/10/8
% 规则：
% (1)正在燃烧的树变成空格位；
% (2)如果绿树格位的最近邻居中有一个树在燃烧，则它变成正在燃烧的树
% (对角线以sqrt(1/2)-0.5的概率引燃)；
% (3)在空格位，树以概率p生长；
% (4)在最近的邻居中没有正在燃烧的树的情况下树在每一时步以概率f(闪
% 电)变为正在燃烧的树。
% 参考文献:
% 祝玉学，赵学龙译，<<物理系统的元胞自动机模拟>>, p23
%%
close all;clc;clear;
figure;
s=300; %树林大小
p=0.02; % 概率p
f=1e-5; % 概率f
p_t=sqrt(1/2)-0.5;
set(gcf,‘DoubleBuffer’,‘on’);
%S=round(rand(s)2);
S=ones(s);
Sk=zeros(s+2);
Sk(2:s+1,2:s+1)=S;
ti=0;
%%
% 白色表示正在燃烧(S中等于2的位置)
% 灰色表示绿树(S中等于1的位置)
% 黑色表示空格位(S中等于0的位置)
C=zeros(s+2,s+2,3);
R=zeros(s);
G=zeros(s);
B=zeros(s);
R(S2)=1;G(S2)=1;B(S2)=1;
R(S1)=0.5;G(S1)=0.5;B(S1)=0.5;
C(2:s+1,2:s+1,1)=R;
C(2:s+1,2:s+1,2)=G;
C(2:s+1,2:s+1,3)=B;
Ci=imshow©;
tp=title(['T = ',num2str(ti)]);
%%
while ti<=3000;
ti=ti+1;
St=Sk;
%%
St(Sk==2)=0; % for rule (1)
%%
Su=zeros(s+2);Sf=Sk;Sf(Sf<1.5)=0;Sf=Sf/2;
Su(2:s+1,2:s+1)=Sf(1:s,2:s+1)+Sf(2:s+1,1:s)+…
Sf(2:s+1,3:s+2)+Sf(3:s+2,2:s+1)+…
Sf(1:s,1:s).(rand(s)<p_t)+Sf(1:s,3:s+2).(rand(s)<p_t)+…
Sf(3:s+2,1:s).(rand(s)<p_t)+Sf(3:s+2,3:s+2).(rand(s)<p_t);
St(Sk1&Su>0.5)=2; % for rule (2)
%%
Se=zeros(s+2);Se(Sk0)=1;
St(2:s+1,2:s+1)=St(2:s+1,2:s+1)+Se(2:s+1,2:s+1).(rand(s)<p);%for rule (3)
%%
Ss=zeros(s+2);Ss(Sk1 & Su0)=1;
St(2:s+1,2:s+1)=St(2:s+1,2:s+1)+Ss(2:s+1,2:s+1).*(rand(s)<f);% for rule (4)
%%
Sk=St;
S_nt=ones(size(Sk(2:end-1,2:end-1)));
S_k=sum(sum(S_nt(Sk(2:end-1,2:end-1)0)));
S_c=sum(sum(S_nt(Sk(2:end-1,2:end-1)1)));
S_s=sum(sum(S_nt(Sk(2:end-1,2:end-1)2)));
list(ti,:)=[S_c,S_s,S_k];
%%
R=zeros(s+2);G=zeros(s+2);B=zeros(s+2);
R(Sk2)=1;G(Sk2)=1;B(Sk2)=1;
R(Sk1)=0.5;G(Sk1)=0.5;B(Sk==1)=0.5;
C(:,:,1)=R;C(:,:,2)=G;C(:,:,3)=B;
set(Ci,‘CData’,C);
%set(tp,‘string’,['T = ',num2str(ti)]);
set(tp,‘string’,['T = ‘,num2str(ti),’,存 = ‘,num2str(S_c),’,烧 = ‘,num2str(S_s),’,空 = ',num2str(S_k)]);
pause(0.1);
end
%%
figure(2)
bar(list,‘stacked’,‘DisplayName’,‘list’);
legend(‘存’,‘烧’,‘空’);
grid on;
